JAG PROJECT

Mail Config Form

The program will check if the MailConfig.properties file exists. If it does, then the Email Application opens right away, if not then we present the form to be filled

The Save button saves and writes the changes to the MailConfig.properties and the Clear button simply erases the text fields on the form. If a connection can be made to the database when clicking on Save, the main application opens.

Menu Bar

The menu item Close performs a Platform.exit() and closes the application. The Add Attachment Menu item opens up the File Explorer and allows users to add regular attachments to the email being created/edited. The Save Attachment Menu Item takes the currently selected table row's email attachments and saves it to the project's root folder (same folder as the pom.xml). Every email's attachments that are sent or received are written to disk into the root folder and stored in the database. The About Menu Item brings you to this current page. The Return to configuration Menu item allows the user to modify the config form and save it once again.

Folder Creation/Deletion

The text field above the two buttons allow the user to input the name of the folder to be created. Users cannot create a folder with the same name as an existing folder. The Delete button will only work if the user clicks on a folder beforehand and will make a pop up if nothing is selected to be deleted. The emails of a corresponding folder are displayed in the table whenever a folder is clicked. Whenever the Inbox folder is clicked, the application updates the database with possible received Emails which may have come in.

Drag and Drop

Users are allowed to drag an email in the table and drop it into any folder except the Draft folder. An alert pop up message is shown if the user tries to drag and drop to/from the Draft folder.

Show Email Details

Whenever an email in the table is clicked, its content along with its attachments are shown in the Form and html editor sections.

Button Functionality

The Send button can send both regular email compositions of draft emails. If a user tries to send an email without any recipients or to a recipient with an invalid format, an error pop up appears with an appropriate message. Email recipients in the TO,CC,BCC can be seperated by semi-colon. ex: recipient1@gmail.com;recipient2@gmail.com

The Save Draft will either update a current draft email if it was clicked on the table or create a completely new draft if no table row was clicked.

The Delete Email button will work on any selected table row that is selected. If nothing is selected, then a pop up error message is shown.

The Reply button takes a selected table row's Sender and subject prefixed with "RE:" and changes are made to the Form.

The Compose button enables the form and editor and clears everything so the user can compose a new email.

The form and html editor are cleared whenever an email is sent/drafted or updated.

Changing folder selections will also clear the form and html editor. Attachments that are added to the email must be inside of the root of the project (same folder as the pom.xml). Emails received that contain attachments are automatically added to the root folder even if it does not exist in the local machine yet.

It is important noting that all incoming and outgoing attachments are saved to the root folder and database regardless of if they are embedded or not.
